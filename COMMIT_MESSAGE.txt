Grafana/ClickHouse: fix stale dashboard data via ingest_time

Summary
- Address dashboards showing no recent data due to historical `ts` values in sensor events.
- Introduce `ingest_time` to represent insertion time and update queries to use it for “current window” views.

Changes
- ClickHouse schema: add `ingest_time DateTime64(3) DEFAULT now()` to `default.sensor_data`; keep `ORDER BY (device_id, ts)` for analytics by event time.
- Consumer: insert explicit `ingest_time = datetime.utcnow()` alongside metrics; `ts` parsed via `datetime.utcfromtimestamp(float(ts))`.
- Grafana datasource provisioning: switch to official plugin v4 keys (`jsonData.host/port/protocol`, `username` in `jsonData`, `password` in `secureJsonData`).
- Dashboards: update queries to use `$__timeFilter(ingest_time)` and latest panels `ORDER BY ingest_time DESC LIMIT 1`.
- README: document rationale, migration steps (ALTER/UPDATE), query patterns, and troubleshooting for plugin config and datasource connectivity.

Operational Notes
- Existing tables require `ALTER TABLE ... ADD COLUMN IF NOT EXISTS ingest_time DateTime64(3) DEFAULT now()` and optional backfill.
- Keep order key as `(device_id, ts)`; use `ingest_time` for dashboard time filters and “latest” reads.
